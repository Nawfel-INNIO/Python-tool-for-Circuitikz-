    CircuitTikZ Visualizer — Project Pipeline

    Overview

    This application lets users build circuit diagrams visually (or write LaTeX manually) and renders them as PDF previews. The pipeline goes from GUI interaction → Python data model → LaTeX code generation → pdflatex compilation → PDF → pixel display.

    Pipeline Steps

    ┌─────────────┐     ┌──────────────┐     ┌─────────────────┐     ┌──────────────┐     ┌────────────────┐
│  User Input  │────▶│  Data Model  │────▶│  LaTeX Source    │────▶│   pdflatex   │────▶│  PDF Preview   │
│  (PyQt5 GUI) │     │  (models.py) │     │  (latex.py)      │     │  (MiKTeX)    │     │  (PyMuPDF)     │
└─────────────┘     └──────────────┘     └─────────────────┘     └──────────────┘     └────────────────┘

    1. User Input — scene.py, panels.py

    • The user places components on a grid canvas (CircuitScene / CircuitCanvas) or writes LaTeX directly in the manual editor tab.
    • Component palette (ComponentPalette in panels.py) lists all available components grouped by category.
    • Property panel (PropertyPanel) allows editing label, value, current, annotation, and color per component.
    • All interactions result in changes to the in-memory Circuit object.

    2. Data Model — models.py

    • Component dataclass: stores kind, label, value, coordinates (x1, y1, x2, y2), current, annotation, color, etc.
    • Busbar dataclass: stores label, y-position, and x-range.
    • Circuit dataclass: holds lists of Component and Busbar objects.
    • COMPONENTS dict: registry mapping component keys (e.g. "R", "C", "trafo") to their CircuitTikZ names, categories, and rendering metadata.

    3. LaTeX Code Generation — latex.py

    • generate_latex(circuit) iterates over all busbars and components in the Circuit and produces a \begin{circuitikz}...\end{circuitikz} snippet.
    • Each component is mapped to its CircuitTikZ \draw ... to[<type>, <options>] ... command using the COMPONENTS registry.
    • Custom colors are converted to \definecolor statements.
    • Node-style components (grounds, power symbols, transistors) use \node[<style>] instead of \draw ... to[...].

    4. LaTeX Wrapping — main.py → latex_template()

    • The snippet is wrapped in a full standalone LaTeX document: latex \documentclass[border=10pt]{standalone} \usepackage[<convention>, siunitx]{circuitikz} \usepackage{xcolor} \begin{document} <snippet> \end{document}
    • The convention (European/American) is selected by the user via a dropdown and passed to the circuitikz package option.

    5. Compilation via pdflatex — main.py → compile_latex()

    • This is where MiKTeX is involved.
    • find_pdflatex() locates the pdflatex executable:
    • Checks hardcoded MiKTeX paths (AppData/Local/Programs/MiKTeX/... and C:/Program Files/MiKTeX/...)
    • Falls back to where.exe pdflatex (Windows) or which pdflatex (Linux/macOS)
    • compile_latex():
    • Creates a temporary directory
    • Writes the full LaTeX document to circuit.tex
    • Calls pdflatex -interaction=nonstopmode -output-directory=<tmp> circuit.tex
    • If circuit.pdf is produced → success; otherwise returns the pdflatex error log

    6. PDF to Pixel Display — main.py → pdf_to_pixmap()

    • Uses PyMuPDF (fitz) to open the compiled PDF.
    • Renders the first page at 200 DPI to a pixel buffer.
    • Converts the buffer to a QPixmap and displays it in a QLabel inside a scroll area.

    7. Cleanup

    • On window close, all temporary directories (containing .tex, .pdf, .aux, .log files) are deleted via shutil.rmtree.

    How MiKTeX Fits In

    MiKTeX provides the pdflatex compiler, which is the core of the rendering pipeline. Without it, the app can still generate LaTeX code but cannot render previews.

    Required LaTeX packages (must be installed in MiKTeX):

    Package Purpose
    circuitikz Provides all circuit component macros
    standalone Produces a tightly-cropped PDF (no margins)
    siunitx Proper formatting of SI units in labels
    xcolor Custom color support for components

    How MiKTeX is called:

    pdflatex -interaction=nonstopmode -output-directory=<tmpdir> <tmpdir>/circuit.tex
    • -interaction=nonstopmode prevents pdflatex from stopping on errors and waiting for user input.
    • The output PDF is read back by PyMuPDF immediately after compilation.
    • MiKTeX's auto-install feature may download missing packages on first use if enabled.

    File Responsibilities

    File Role
    main.py Entry point, MainWindow, LaTeX template, compilation pipeline (find_pdflatex, compile_latex, pdf_to_pixmap)
    models.py Data model (Component, Busbar, Circuit) and COMPONENTS registry
    latex.py LaTeX code generation from Circuit + JSON save/load
    scene.py Canvas interaction (placement, wiring, selection, drag, undo/redo)
    panels.py Component palette tree and property editor panel

    Data Flow Summary

    User clicks grid          → Component added to Circuit (models.py)
Circuit changes            → generate_latex() produces LaTeX snippet (latex.py)
Auto-build or Ctrl+Enter   → compile_latex() writes .tex, calls pdflatex (MiKTeX)
pdflatex produces .pdf     → pdf_to_pixmap() renders it via PyMuPDF (fitz)
QPixmap displayed           → User sees the rendered circuit
